# 프로젝트 요구사항 정의서 (PRD)

## 1. 프로젝트 목적
사용자가 미국 및 국내(한국) 주식 시장의 핵심 지수와 주도 테마를 한눈에 파악할 수 있는 종합 주식 대시보드 애플리케이션 개발.

## 2. 주요 기능 및 요구사항

### 2.1 공통 기능
- **미니 차트 표시:** 각 인덱스 지수는 미니 차트 형태로 제공.
  - 당일 시가와 종가의 변화 흐름을 직관적으로 확인할 수 있도록 하루치 데이터 라인 차트로 표시.
- **등락률 표시:** 모든 지수 및 종목의 등락률은 당일 종가 기준으로 표시.

### 2.2 미국 시장 정보
- **제공 데이터 (인덱스/지표):** 
  - S&P 500, 나스닥, 다우존스, 나스닥 100, 러셀 2000, 필라델피아 반도체 지수
  - WTI 원유 가격, VIX(변동성) 지수
- **섹터 및 테마 시황:**
  - Finviz 데이터를 활용하여 미국 시장의 주요 섹터 및 하위 테마의 당일 등락률을 5-컬럼 보드 형태로 시각화.
  - 이름, 등락률 순 정렬 기능 및 요약 보기/전체 보기 토글 기능 제공.
- **데이터 소스:** FinanceDataReader, Finviz (테마/섹터 데이터)
- **데이터 소스:** FinanceDataReader

### 2.3 국내 시장 정보
- **주요 지수:** 코스피(KOSPI), 코스닥(KOSDAQ)
- **파생 상품:** 야간선물 (전일 기준 종가/등락 표시)
- **당일 주도 테마 (시장 주도주) 분석:**
  - **표시 항목:** 테마명 | 종목명 | 현재가 | 등락률 | 거래대금
  - **주도 테마 파악 로직:** 
    - 당일 거래대금(총액)
    - 테마 소속 종목들의 평균 등락률
    - 상승을 견인한 테마(종목)의 갯수
    - 위 요소들을 종합해 핵심 테마 도출.
  - **분석 설명:** 대시보드 내에 '주도 테마를 어떻게 파악했는지'에 대한 로직 설명을 텍스트로 제공 (사용자 신뢰도 향상).

## 3. 비기능 요구사항
- **UI/UX:** 반응형 대시보드. 심플하고 직관적인 레이아웃. 상단은 '미국 시장', '한국 시장', '차트' 탭으로 분리하여 사용성을 높임.
- **성능:** API 호출의 지연을 최소화하기 위해 적절한 캐싱 기법(cachetools 또는 st.cache_data) 적용.

## 4. 추가 기능 요구사항 (5/1 추가개발)
### 4.1 트레이딩뷰 위젯 차트
- **기능:** 트레이딩뷰(TradingView) 임베디드 코드를 활용하여 메인 페이지 내 '차트' 탭에 다중 종목 차트(Area 형식) 제공.
- **심볼 서치:** 
  - 한글명, 영문명, 티커명으로 검색 가능.
  - 검색된 결과를 선택하여 차트에 동적으로 종목 추가 및 삭제 가능.
  - 심볼 리스트는 별도의 DB 없이 API(Yahoo Finance 등)를 활용하여 실시간 검색 지원.
  - 페이지 리프레시 없이 심볼 추가 시 바로 차트에 반영.
